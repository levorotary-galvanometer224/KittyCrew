## KittyCrew を選ぶ理由

多くの AI ツールは、無機質な機能パネルのように見えます。KittyCrew はそれを、仲間たちが一緒に暮らすあたたかい家に変えます。

- Claude Code、Codex、GitHub Copilot を個性の違う仲間として一緒に置けます。
- 最大 5 メンバーの小さな crew としてペットや仲間をまとめられます。
- 各メンバーごとにモデル、作業ディレクトリ、スキルの許可範囲を設定できます。
- メンバーごとの会話履歴、記憶の流れ、ストリーミング応答を 1 つの UI で見守れます。
- ローカルファーストで動作し、利用可能な provider を手元の環境から自動検出します。

KittyCrew は、AI を道具ではなく、そばにいる存在として感じたい人のためのツールです。ローカル制御も、心地よい UI も手放しません。

## 主な機能

### あたたかいマルチペットホーム

- 1 つの Web アプリで複数の crew を作成・管理できます。
- crew ごとに、AI ペットや仲間たちの小さな家族を作れます。
- 各メンバーの応答や様子をそれぞれ見守れます。

### 個性ある AI 仲間たち

- Claude Code、Codex、GitHub Copilot のメンバーをサポートします。
- ローカルの provider CLI を実行時に自動検出します。
- 将来の会話に適用されるメンバー単位のモデル設定を保持できます。

### ローカルで安心

- 各メンバーは独立した永続セッション状態を持ちます。
- 各メンバーに別々の作業ディレクトリを設定できます。
- すべてを開放するのではなく、スキルへのアクセスをメンバーごとに制限できます。

### 愛着が湧く UI

- 猫テーマの crew カードとアバター選択。
- インラインでの名前変更、削除、キュー投入、キャンセル操作。
- 長く過ごす会話のための拡大メンバービュー。

## クイックスタート

### 1. インストール

```bash
python -m pip install -e .
```

### 2. 起動

```bash
kittycrew
```

デフォルトでは Web UI が [http://127.0.0.1:8731](http://127.0.0.1:8731) で起動します。

リポジトリのルートから直接実行することもできます。

```bash
PYTHONPATH=src python -m kittycrew
```

## 仕組み

KittyCrew は FastAPI ベースの Web アプリと、`a2a-sdk` で公開される provider アダプターを組み合わせています。

- Web UI が crew、member、チャット状態、ローカル永続化を管理します。
- Provider アダプターが Claude Code、Codex、GitHub Copilot の CLI を橋渡しします。
- 各メンバーは独立した実行設定を持つ分離セッションに対応します。
- ストリーミング出力は UI にそのまま反映され、仲間がそこにいる感覚を保てます。

## ユースケース

- 個性の違う AI ペットや仲間たちを小さな家族として迎える。
- 日常の付き添い、創作、会話などで crew を分けて過ごす。
- メンバーごとに部屋、モデル、許可スキルを分けて育てる。
- ツール画面というより、いつでも開いておける AI の居場所として使う。

## ロードマップ

- 同じ crew モデルのまま、より多くの仲間タイプに対応する。
- メンバー同士の関わり方や日常の体験を強化する。
- 各メンバーの履歴や状態をもっと自然に見守れるようにする。
- セットアップとオンボーディング体験を改善する。

## 補足

- KittyCrew はメンバーごとの会話履歴を保存し、次回以降のターンに最近の文脈を再利用します。
- 利用するには、対応する provider CLI と `a2a-sdk` が現在の環境で使える必要があります。
- Provider の利用可否は実行時に検出されるため、ローカル環境が不完全でも可能な範囲で動作します。
